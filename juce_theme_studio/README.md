# JUCE Theme Studio

Standalone Python desktop application for designing, previewing, editing, and exporting JUCE plugin and app themes using sprite-sheet assets.

## Requirements

- Python 3.11+
- macOS, Linux, or Windows (developed and tested on macOS-compatible Qt stack via PySide6)

## Install

```bash
cd juce_theme_studio
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

## Launch

```bash
juce-theme-studio
# or
python -m juce_theme_studio.app.main
```

## Load a JUCE Project

1. Click **Open Project** in the toolbar.
2. Select your JUCE project root (folder containing `.jucer`, `CMakeLists.txt`, or `Source/`).
3. The app creates a non-destructive editor folder:

   ```text
   .juce_theme_studio/
     theme_project.json
     screens/
     assets/
     exports/
     backups/
     logs/
   ```

4. Detected `juce::Component` screens appear in the left **Screens** list. Use **New Screen** for manual layouts.

Your original source and assets are never modified.

When you reopen a project, saved control positions and sizes are restored from `theme_project.json`. You can drag controls to move them and use the resize handle unless they are **locked** (Layers panel) or **Preview Mode** is enabled.

## Asset Library

Images under your JUCE project (`Resources/`, `BinaryData/`, etc.) are detected on open but are **not** copied automatically unless you confirm the prompt (when the library is empty) or click **From Project** in the Asset Library section.

- **Import Asset** — pick any file from disk.
- **From Project** — copy all detected project images into `.juce_theme_studio/assets/` (skips files already imported).
- **Import Sprite Sheet** — import a strip/grid with optional per-frame slicing.

## Import Sprite Sheets

1. Click **Import Sprite Sheet** in the Asset Library.
2. Select a PNG/JPG/WEBP strip or grid image.
3. A copy is stored in `.juce_theme_studio/assets/`.
4. Select the asset, choose a control type (Knob, Button, Meter, etc.), and click **Add Control**.
5. Edit frame count, dimensions, and layout in the **Properties** panel → **Sprite Sheet** section.

## Create Controls

1. Select a screen in the sidebar.
2. Pick a control type from **Control Palette**.
3. Optionally select an asset first.
4. Click **Add Control**.
5. Drag to move, drag the bottom-right handle to resize.
6. Use the **Layers** panel for z-order, visibility, and lock.
7. Edit X/Y/Width/Height numerically in **Properties**.

## Preview Behavior

- Toggle **Preview Mode** in the toolbar to disable editing and simulate interaction.
- Use **Preview Controls** (right sidebar) for button state (normal/hover/active/disabled).
- Adjust **Value** and **On/Active** in Properties for knobs, meters, sliders, and LEDs.

Coordinates use a top-left origin matching JUCE `setBounds(x, y, w, h)`.

## Map Controls to JUCE Code

In **Properties → JUCE Mapping**:

- **JUCE class** — e.g. `juce::Slider`
- **C++ variable** — e.g. `gainSlider`
- **Parameter ID** — for knobs/sliders

The scanner auto-detects components from `Source/` and suggests mappings. Manual override is always available.

## Export

1. Click **Save**, then **Export**.
2. Review the **Validation** tab for warnings/errors.
3. Generated files are written to `.juce_theme_studio/exports/`:

   - `ThemeLayout.json`
   - `ThemeAssets.h` / `.cpp`
   - `ThemeLookAndFeel.h` / `.cpp`
   - `GeneratedThemeComponents.h` / `.cpp`
   - `assets/` (copied images)

4. Previous exports are backed up to `.juce_theme_studio/backups/`.
5. Generated C++ files are marked with `// GENERATED BY JUCE Theme Studio`.

Integrate exported files into your JUCE project manually or via your own build step. Existing source files are never overwritten by the exporter.

## Git Commit

1. Click **Commit** (only enabled for Git repositories).
2. Review changed files under `.juce_theme_studio/`.
3. Preview diffs.
4. Edit the commit message.
5. Check the confirmation box and press OK.

**The app never commits automatically.** Unrelated repo changes trigger a warning.

## Layout Tools

Use the **Layout** menu or drag controls on the canvas:

- Smart snap guides appear when edges/centers align with other controls or the canvas
- **Layout → Align** — left, center, right, top, middle, bottom
- **Layout → Distribute** — even horizontal/vertical spacing (3+ controls)
- **Screen Settings** panel — edit canvas width/height per screen
- **Settings** toolbar — grid size, snap toggle, C++ export namespace

## JUCE Auto-Mapping

On project open, the scanner adds placeholder controls for detected C++ members (`gainSlider`, etc.) with JUCE class and variable mapping pre-filled. Use **Project → Sync JUCE Mappings** or **Rescan Project** to refresh.

## Optional extras

Install advanced parsing and vision backends:

```bash
pip install -e ".[full]"
# or individually:
pip install -e ".[parsing]"   # tree-sitter-cpp
pip install -e ".[vision]"    # opencv-python-headless
pip install -e ".[clang]"     # libclang
```

- **tree-sitter-cpp** / **libclang** — precise C++ component scanning (falls back to regex)
- **OpenCV** — improved sprite sheet frame auto-detection

## Drag-and-drop

Drag assets from the **Asset Library** onto the canvas:

- **Empty area** — creates a new control (type from Control Palette) at the drop position.
- **Existing placeholder** — a **Link Asset to Control** dialog appears showing the C++ variable, JUCE class, and asset name. Choose **Link** or **Cancel**.

## Sprite sheet → multiple library assets

When importing a sprite sheet, the configuration dialog offers:

- **Slice all frames into asset library** — each frame saved as its own PNG (`name_frame_00`, `name_frame_01`, …).
- **Also keep full sprite sheet in library** — retains the master strip for strip-based controls.

Use sliced frames for single-state images (LEDs, decals); keep the full sheet for knobs and meters.

## Theme diff

**Project → Theme Diff** compares manifests or your current project against the latest export backup.

## Live JUCE preview

1. Build the companion app: `examples/juce_live_preview/` (requires JUCE on macOS).
2. Enable **Live JUCE Preview → Auto-export on edit** in the right panel.
3. Point to your `JuceLivePreview` binary for runtime rendering of exported `ThemeLayout.json`.

## Limitations

- Native JUCE preview requires building the companion example with JUCE installed.
- libclang needs Xcode CLT paths on macOS for best results.
- SVG rendering depends on Qt/Pillow capabilities.
- WEBP requires Pillow built with WEBP support.

## Safety

- Does not delete or rewrite original JUCE project files
- Does not auto-commit
- Uses relative paths in the manifest where possible
- Logs to GUI and `.juce_theme_studio/logs/studio.log`
